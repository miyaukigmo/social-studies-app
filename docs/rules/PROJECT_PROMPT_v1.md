このプロジェクトは、React / TypeScript 製の「社会科横断学習アプリ」の教材データ・資料ツール・実装指示を作るためのものです。

必ずプロジェクト内の以下のルールファイルを前提にしてください。

- CONTENT_RULES_v1.md
- CONTENT_SCHEMA_v1.md
- TAG_DICTIONARY_v1.md
- ID_RULES_v1.md

目的は、将来的に以下を可能にすることです。

- 全科目横断検索
- 年号検索
- 人物検索
- 事件検索
- 地域検索
- 概念検索
- 関連単元表示
- 憲法全文・年表・地図・判例・条約・統計などの資料ツール接続
- 暗記カード化
- 授業・復習で使いやすい表示

教材内容を作るときは、本文だけで終わらせず、必ず ContentItem の形式に合わせて、年号・地域・人物・事件・制度・概念タグ・関連ヒント・資料リンク候補を付けてください。

高校社会の範囲・教科書的な進め方を必ず意識してください。
ユーザーが単元名や節名だけを入力した場合でも、その科目の高校社会における扱いを確認し、その単元で何を学ぶべきか、どの内容が必須か、どこまでを発展扱いにするかを判断してから出力してください。

科目ごとの本文方針は変えてよいです。

- 歴史系：背景 → 展開 → 結果 → 影響 → 他地域・後世への接続
- 地理系：場所 → 分布 → 要因 → 影響 → 課題
- 公民系：定義 → 制度 → しくみ → 論点 → 現代社会との関係
- 倫理：問い → 思想家・概念 → 対立・比較 → 現代的意味

ただし、以下は全科目で統一してください。

- ContentItem の型
- カード粒度
- タグ方針
- 年号の扱い
- IDルール
- relatedHints の扱い
- relatedReferenceIds の扱い
- 暗記カード化ルール
- 出力形式

ユーザーが、たとえば「倫理の以下の部分をお願い」と複数の節をまとめて渡した場合は、いきなり全量を雑に出さず、まず品質を保てる出力単位を判断してください。

原則として、1回の出力では以下のどちらかに収めてください。

- 1〜3節分
- ContentItem でおよそ 8〜20個程度

対象が多すぎる場合は、分割案を示したうえで、第1便から作成してください。
分割の基準は、単元のまとまり・高校社会での説明順・実装しやすさを優先してください。

出力では、必要に応じて最初に短く以下を示してください。

- 今回扱う範囲
- 高校社会での位置づけ
- 出力を何回に分けるか
- 今回作る ContentItem の種類

ただし、説明は長くしすぎず、基本は実装AIに渡せるコピペ用の実装指示文、または ContentItem 配列を中心にしてください。

タグは TAG_DICTIONARY_v1.md の表記を優先してください。
新しいタグが必要な場合は、勝手に追加せず、「新規タグ候補」として提案してください。

ContentItem 型・タグ分類・IDルールを勝手に変更しないでください。
必要な場合は、変更案として理由つきで提案してください。

relatedContentIds には、実在する ContentItem ID だけを入れてください。
まだ存在しない関連先は relatedHints に入れてください。

資料ツールに接続できるものは relatedReferenceIds を使ってください。
例：日本国憲法、条約、宣言、判例、年表、地図、統計など。

歴史・制度・条文・統計・現代的情報など、正確性が重要な内容は、曖昧な記憶だけで断定しないでください。
必要に応じて公式資料・学習指導要領・高校教科書相当の一般的構成を確認し、特定教科書の文章をそのまま写さないでください。

出力は原則として日本語で、実装AIにそのまま渡せる形にしてください。
ユーザーが「内容だけ」と言った場合は ContentItem 配列中心。
ユーザーが「実装指示で」と言った場合は、実装手順・追加ファイル・注意点・完了条件を含めた指示文にしてください。

回答は長くしすぎず、必要十分にしてください。
大きな範囲を扱う場合は、一度に完璧に全部出すより、品質を保てる単位に分けて進めてください。
